## Часть 2. Разбор приложения «Галактики»

### Скриншот результата  
<img width="985" height="266" alt="image" src="https://github.com/user-attachments/assets/af7ca52f-c668-47cf-b14f-c3dfcb64fda4" />

### Краткое описание  
Программа выводит таблицу с данными о галактиках: название, удалённость (в мегасветовых годах), тип строения. В изначальной версии присутствовало два дефекта, которые я выявил и устранил.

### Найденные дефекты  

**Дефект 1 — несоответствие типа данных**  
Свойство `GalaxyType` ошибочно имело тип `object`, хотя должен был быть `GType`. Из-за этого вместо читаемого названия типа печаталось полное имя класса.

**Дефект 2 — ошибочный символ в конструкции switch**  
В теле конструктора `GType` в метке `case` стояла маленькая буква `'l'`, а требовалась большая `'I'`. Из-за этой опечатки значение `Irregular` никогда не присваивалось.

### Ход отладки  

| Инструмент | Мои действия |
|------------|---------------|
| **Breakpoints** | Установил их на строке вывода в цикле `foreach` и на входе в конструктор `GType`. |
| **Data Tips** | Навёл указатель на `theGalaxy.GalaxyType`, развернул подсказку и заметил внутреннее поле `MyGType` — это натолкнуло на мысль о проблеме с типом. |
| **Step Into (F11)** | Вошёл внутрь конструктора `GType`, чтобы в деталях проверить логику переключателя. |
| **Edit and Continue** | Прямо в процессе отладки изменил обращение на `GalaxyType.MyGType` — не пришлось перезапускать программу. |
| **Restart (Ctrl+Shift+F5)** | После корректировки типа запустил приложение заново, чтобы убедиться в исправлении. |

## Часть 3. Разбор приложения «Буквы»

### Скриншот результата  
<img width="977" height="250" alt="image" src="https://github.com/user-attachments/assets/a061cd52-b810-4b2a-aa52-f4352412270c" />

### Краткое описание  
Утилита перебирает символы из массива `letters`, постепенно наращивая строку `name`, и каждый раз вызывает метод `SendMessage`. Этот пример удобен для тренировки работы с отладчиком.

### Какие средства отладки я задействовал  

| Инструмент | Как я работал с ним |
|------------|----------------------|
| **Breakpoints** | Поставил точку на строке `name += letters[i]` внутри цикла. |
| **Data Tips** | Через всплывающую подсказку просмотрел содержимое массива `letters` (там 10 элементов); также следил за изменением строки `name` после каждой итерации. |
| **Run to Click** | Кликнул по строке `Console.WriteLine` в методе `SendMessage` — отладчик сразу перешёл туда, пропустив промежуточные шаги. |
| **Окно Autos** | Изучал переменные, которые система сама подхватывает из текущей и предыдущей строк кода. |
| **Окно Locals** | Проверял текущие значения `letters`, `name`, вспомогательной переменной `a` и счётчика `i`. |
| **Окно Watch** | Вручную добавил переменную `name` в контрольное окно — наглядно видел, как она увеличивается с каждым проходом цикла. |
| **Call Stack** | Отследил последовательность вызовов `Main → RunLetters → SendMessage`; двойным щелчком переходил к нужной строке в коде. |
| **Restart** | Несколько раз перезапускал программу с разными точками останова, чтобы протестировать сценарии. |
